# ft_printf

## Description

`ft_printf` is a partial reimplementation of the C standard library function `printf`. The goal of this project is to understand how variadic functions work in C, how a format string is parsed, and how each conversion specifier translates into output.

The implemented conversion specifiers are:

- `%c` — prints a single character
- `%s` — prints a string
- `%d` / `%i` — prints a signed decimal integer
- `%u` — prints an unsigned decimal integer
- `%x` / `%X` — prints an integer in hexadecimal (lowercase / uppercase)
- `%p` — prints a pointer address in hexadecimal
- `%%` — prints a literal `%`

---

## Algorithm and Data Structure

### Parsing the format string

The core of `ft_printf` is a loop that reads the format string character by character:

- If the current character is not `%`, it is printed directly.
- If it is `%`, the next character is read to identify the conversion specifier, and the corresponding function is called.

This approach is a simple **linear parser** — no tree, no stack, just sequential reading. It is sufficient for the subset of specifiers required by this project.

### Variadic arguments

The function uses the `stdarg.h` macros (`va_start`, `va_arg`, `va_end`) to access the variable argument list. Each time a specifier is encountered, `va_arg` is called with the correct type to retrieve the next argument.

### Integer conversion

Integers are printed recursively: the function divides the number by its base, recurses on the quotient, then prints the remainder. This naturally produces digits in the correct order without needing a buffer or a reversed string.

### Unsigned integers and `%u`

Since `%u` expects an unsigned value, the parameter is declared as `unsigned int`. Any negative `int` passed by the caller is automatically reinterpreted as its unsigned equivalent by the C type system — no manual conversion needed.

### Pointer addresses and `%p`

A `void *` pointer is cast to `unsigned long` to obtain a numeric representation of the address. The value is then printed in hexadecimal with a `0x` prefix. A `NULL` pointer prints `(nil)` as per standard behavior.

### Hexadecimal conversion

The function uses a character string as a base (`"0123456789abcdef"` or `"0123456789ABCDEF"`) and applies the same recursive digit-extraction logic as the decimal printer, with base 16.

## Instructions

### Compilation

Clone the repository and run:

```bash
make
```

This will produce `libftprintf.a`.

### Using in your project

Include the header and link against the library:

```c
#include "ft_printf.h"
```

```bash
cc your_file.c -L. -lftprintf -o your_program
```

### Cleaning

```bash
make clean   # removes object files
make fclean  # removes object files and the library
make re      # full recompilation
```

## Usage Example

```c
#include "ft_printf.h"

int main(void)
{
    ft_printf("Hello, %s!\n", "world");
    ft_printf("Decimal: %d\n", -42);
    ft_printf("Unsigned: %u\n", -1);
    ft_printf("Hex: %x\n", 255);
    ft_printf("Pointer: %p\n", (void *)0);
    return (0);
}
```

Expected output:

```
Hello, world!
Decimal: -42
Unsigned: 4294967295
Hex: ff
Pointer: (nil)
```
